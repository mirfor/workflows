## Documentation Style Guide

When generating Markdown documentation:

- Write in the style of technical reference documentation.
- Include only substantive information.
- Do not add introductions, conclusions, or stylistic commentary.
- Do not use filler phrases such as:
  - "Below is..."
  - "It is worth noting..."
  - "Keep in mind..."
  - "The key point is..."
- Avoid repetition.
- Each sentence must add new information.
- Prefer bullet lists and tables over narrative text.
- Use short paragraphs.
- Maintain consistent heading hierarchy.
- Maximize information density.
- Minimize word count.

After generating any document, perform a second pass and remove every sentence whose removal does not reduce the informational content.
